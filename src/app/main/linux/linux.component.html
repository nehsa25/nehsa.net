  • Common Commands
    • Working with Apt
      • Install a package: sudo apt-get install <PACKAGE>
        NOTE: Despite what my fingers muscle memory says, apt-get can just be apt. i.e. sudo apt install <PACKAGE>
      • Search for a package: apt-cache search <PACKAGE>
      • Uninstall a package: sudo apt-get remove <PACKAGE>
      • Update the package list: sudo apt-get update
      • Upgrade the system: sudo apt-get upgrade
      • Upgrade the system (including kernel): sudo apt-get dist-upgrade
    • Working with Snap To configure Snap, we need to install the snapd package: sudo apt-get install snapd

      • To install a package using Snap, use the following command: sudo snap install <PACKAGE>
      • To uninstall a package using Snap, use the following command: sudo snap remove <PACKAGE>
      • To list installed Snap packages, use the following command: snap list
      • To update all Snap packages, use the following command: sudo snap refresh

  • Chmodding and Chowning
    • Chmodding Linux uses an 3-digit octal number to represent the permissions of a file or directory.

      The numbers work as follows:
      • 4: Read
      • 2: Write
      • 1: Execute


      Some examples:
      • 444 indicates: Owner, Group, and Others have read permissions only.
      • 755 indicates: Owner has read, write, and execute permissions; Group has read and execute permissions; Others have read and execute permissions.
      • 777 indicates: Owner, Group, and Others have read, write, and execute permissions. Also called "The full burrito." (not really, I just made that up)


      To change the permissions of a file or directory, use the following command: chmod <PERMISSIONS> <FILE>
      e.g. chmod 777 example_file

      Or, if you're ganster, you can use the symbolic method:
      1. chmod g+x <FILE> - Modify the file allowing the group to execute it.
      2. chmod u+r <FILE> - Allow the owner to read it.
      3. chmod o-w <FILE> - Prevent others from writing to it.
      4. chmod a=rwx <FILE> - Give everyone full access to the file.
      5. chmod u=rw,go=r <FILE> - Give the owner read and write permissions, and everyone else read permissions.
    • Chowning Files and directories in a Linux environment have two main attributes: the owner and the group. The owner is the user who created the file or directory, and the group is a collection of users who have access to the file or directory. The owner and group can have different permissions, which can be set using the chown command.

      To change the owner of a file or directory, use the following command: chown <USER> <FILE>

      Examples:
      • chown root example_file - Change the owner of the file to root.
      • chown root:web example_file - Change the owner of the file to root and the group to web.

  • Apt Vs. Snap Snap is a package manager developed by Canonical (makers of Ubuntu) that allows for the installation of packages in a sandboxed environment. This means that the package is isolated from the rest of the system, which can help prevent conflicts and security issues. Snap packages are also self-contained, meaning that they include all of the dependencies that they need to run. This can make them larger than traditional packages, but it also means that they are more likely to work on different distributions of Linux.

    Snap packages are containerized.
  • Enabling Snap on Mint Linux Due to the proprietary nature of Snap, Mint Linux decided to prevent it's use by default. To enable Snap, we need to:

  • Configure Bluetooth I use a Bluetooth headset with my Linux machine using a USB adapter: TP-LINK UB500.

    Steps to configure Bluetooth:
    • Plug in the USB adapter
    • Run lsusb to see if the adapter is recognized (we should see something like:
      Bus 001 Device 006: ID 2357:0604 TP-Link TP-Link UB500 Adapter)
    • Run sudo apt-get install bluez bluez-tools bluez-firmware, this will install the necessary Bluetooth packages
    • Run sudo systemctl start bluetooth to start the Bluetooth service
    • Run sudo systemctl enable bluetooth to enable the Bluetooth service on boot
    • Restart the bluetoot service: sudo systemctl restart bluetooth
    • Run sudo systemctl status bluetooth to check the status of the Bluetooth service
    • Launch the Cinnamon Bluetooth applet (called blueman-manager)
    • Run scan on to scan for Bluetooth devices
    • Run pair <MAC ADDRESS> to pair with the device
  • Configure Python Python packages Vs. system packages
    While it used to be common to use Python's built-in package manager, pip, it's now expected to use the system package manager instead when using Python in a Linux environment. This allows the Python packages to be installed and maintained by the system package manager.

    To install a package to a specific Python version, use the following command: python3.11 -m pip install <PACKAGE>
    (or pip install <PACKAGE>)

    To install a package system-wide, use the following command: sudo apt install python3.11-<PACKAGE>
  • Configure SSH
    • Install It: <USER>@<HOST>:~$ sudo apt install openssh-server
    • Start It: <USER>@<HOST>:~$ sudo systemctl start ssh
    • Enable It (Start on boot): <USER>@<HOST>:~$ sudo systemctl enable ssh
    • Check Status: <USER>@<HOST>:~$ sudo systemctl status ssh
    • Configure it: <USER>@<HOST>:~$ sudo vim /etc/ssh/sshd_config

      # Common options:
      # doesn't allow root login: PermitRootLogin no
      # doesn't allow password login (must have ssh key): PasswordAuthentication no
      # doesn't allow empty passwords: PermitEmptyPasswords no
      # doesn't allow X11 forwarding: X11Forwarding no
      # doesn't allow agent forwarding: AllowAgentForwarding no
      # doesn't allow TCP forwarding: AllowTcpForwarding no
      # Restrict access to specific users (delimited by space): AllowUsers <USER>
      # Restrict access to specific groups (delimited by space): AllowGroups <GROUP>
      # Restrict access to specific IP addresses: AllowUsers <USER>@<IP>
      # Restrict access to specific IP ranges: AllowUsers <USER>@<IP>/<CIDR>
      # Restrict access to specific ports: AllowUsers <USER>@<IP>:<PORT>
      # Ban after 3 failed login attempts: MaxAuthTries 3 (/etc/ssh/sshd_config)
    • Restart It: <USER>@<HOST>:~$ sudo systemctl restart ssh

        •     [homes]
    comment = Home Directories
    browseable = yes
    read only = no
    create mask = 0775
    directory mask = 0775

  • Setup Docker
    1. Add the Docker repo:
      1. Create folder for Docker Apt repo: sudo mkdir -p /etc/apt/keyrings
      2. Download the Docker GPG key: curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
        • Ubuntu/Mint:
          echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu jammy stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
          Linux Mint 22 codename "wilma" should point to the Ubuntu 22 codename "jammy"
        • Debian:
          echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/debian bookworm stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
          Debian 12 is "bookworm"
    2. Update apt: sudo apt-get update
    3. Install dependancies: sudo apt-get install docker-ce docker-ce-cli containerd.io docker-compose-plugin
    4. Install the dep package: sudo apt-get install ./docker-desktop-<version>-<arch>.deb
    5. Configure Pass (required to be able to login)
      • gpg --generate-key
      • pass init <KEY>
    6. At this point, we should be able to start the docker-desktop
