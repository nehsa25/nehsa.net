Non-sepcific Cloud Lingo:
Infrastructure as a ServiceIaaS
An IaaS is a cloud service that provides virtualized computing resources over the internet.
Software as a ServiceSaaS
SaaS is a software distribution model in which a third-party provider hosts applications and makes them available to customers over the internet.
Platform as a ServicePaaS
This differs from similar services like IaaS or SaaS in that it provides a platform for developers to build applications on.
Integration Platform as a ServiceiPaaS
Integration Platform as a Service is a suite of cloud services and tools thatthat further streamline deployment by providing pre-built connectors to common SaaS applications and on-premises systems and tools for building custom integrations. It also provides a centralized console for monitoring and managing integrations and often includes features for data mapping, transformation, and error handling.

A use case for iPaaS would be to connect a CRM system to an ERP system. This would allow a salesperson to see the status of an order in real-time, without having to switch between systems.

Examples include: MuleSoft, Dell Boomi, and Workato.
Function as a ServiceFaaS
FaaS is a category of cloud computing services that provides a platform allowing customers to develop, run, and manage application functionalities without the complexity of building and maintaining the infrastructure typically associated with developing and launching an app.


Methods to host your application:
Service Offering Strengths Weaknesses Examples When would you use?
SaaS Easy to use, no maintenance Less control, the provider manages the software Google Docs, Office 365, Salesforce When you have a self-contained solution already and you wish to host it in the cloud.
PaaS Easy to deploy, no maintenance Less control, the provider manages the platform Heroku, Google App Engine, Azure App Service When you need a platform to build your application on and you don't want to manage it. Differs from SaaS because you're building the application within the platform.
FaaS Easy to deploy, no maintenance Less control, the provider manages the platform Google Cloud Functions, AWS Lambda, Azure Functions When you need to run a small piece of code in response to an event. For example, you could use a function to resize an image when it's uploaded to a storage bucket.
IaaS Most control, most flexibility Most maintenance, most expensive EC2, Azure VMs, Google Compute Engine When you need full control over the infrastructure and you're willing to manage it.
Serverless Easy to deploy, no maintenance Less control, the provider manages the platform Azure Static Web Apps, AWS Lambda, Google Cloud Functions When you need to run a small piece of code in response to an event. For example, documentation is a good use case for serverless architecture.
