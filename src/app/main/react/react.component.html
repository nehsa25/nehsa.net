React Web Development
I am a much stronger front-end developer using Angular+Material than React. This page is my study of React.

My first React site will use Vite, a modern build tool, Remix, a framework that supports server-side rendering and static content generation, and Tailwind CSS, a CSS frameword that provides utility classes such as flex, grid, and spacing. e.g. <span class="code">class="flex justify-center"</span> Here is the test application I build: react-vite-app

I will make another application after that using next.js as it has the largest adoption.
  • Installation The first choice encountered when building a new React app is the toolchain to use.
    Toolchain Pros Cons Cost License Vendor Community Maturity
    create-react-app Fast, easy to use, and well-documented Less flexible than other options Free MIT Facebook Very large High
    Next.js Server-side rendering, static site generation, and API routes More complex than create-react-app Free MIT Vercel Large High
    Gatsby Static site generation, server-side rendering, and a rich plugin ecosystem More complex than create-react-app Free MIT Open-source Large High
    Vite Fast, modern, and flexible Less mature than create-react-app Free MIT Open-source Medium Medium


    In addition to toolchains, there are many libraries and frameworks to consider such as Remix and SWC.
    * Remix is a full-stack framework that includes server-side rendering, data fetching, and routing.
    * SWC is a fast JavaScript/TypeScript compiler allowing for a better developing environment.


    As a first attempt at React, I'm going to use Vite & Remix for the most cutting edge experience.

    • Install Vite: npx create-remix@latest
      * Preferred over "npm init vite," more configuration required this method.
    • Install the dependencies: npm install
    • Install Tailwind CSS and PostCSS
      1. npm install tailwindcss postcss autoprefixer - Install the necessary packages
      2. npx tailwindcss init -p - Create the configuration file which allows us to configures such things as the color palette and the spacing scale
      3. Edit the tailwind.config.js file to add the colors and spacing scale 
        module.exports = {
    mode: 'jit',
    purge: ['./app/**/*.jsx'],
    darkMode: false, // or 'media' or 'class'
    theme: {
        extend: {
            colors: {
                'nehsa-blue': '#1e3a8a',
                'nehsa-green': '#00ff00',
                'nehsa-red': '#ff0000',
                'nehsa-yellow': '#ffff00',
                'nehsa-orange': '#ff6600',

                'nehsa-pink': '#ff00ff',
                'nehsa-brown': '#663300',
                'nehsa-gray': '#666666',
                'nehsa-black': '#000000',
                'nehsa-white': '#ffffff',
            },
            spacing: {
                '1': '0.25rem',
                '2': '0.5rem',
                '3': '0.75rem',
                '4': '1rem',

    }
      4. Edit the postcss.config.js file to include tailwindcss and autoprefixer 
        module.exports = {
    plugins: [
        require('tailwindcss'),
        require('autoprefixer'),
    ],
};
      5. Edit the app/assets/tailwind.css file to include the tailwind directives 
        @tailwind base;
@tailwind components;
@tailwind utilities;
      6. Include the tailwind.css file in the app/root.jsx file 
        import './app/assets/tailwind.css';
      7. Verify the TailWind CSS installation with this CSS: 
        <div class="bg-nehsa-blue text-nehsa-white p-4 m-4">Hello World</div>
      8. Build
    • Run the development server: npm run dev
    • Open the app in a browser: http://localhost:5173
  • Basics

    • Important Files and Directories

      app
      Top-level directory
      app/routes
      Contains the routes for the application
      app/root.jsx
      The root route, the first page loaded
      entry.client.tsx
      Client-side entry point, loads the app
      entry.server.tsx
      Server-side entry point, loads the app
      tailwind.css
      Global CSS, Tailwind is a utility-first CSS framework meaning it provides low-level utility classes to build custom designs
      app/assets
      Contains images, fonts, and other assets
    • Adding a global stylesheet:
      1. Edit app/root.tsx
      2. import './app/assets/tailwind.css';
