Coding Conventions

  • Constants: must be all upper case, with multiple words separated by '_' final int ITEMS_IN_BASKET = 42;
  • Classes: must start with an upper case letter, and use CamelCase. class LargeSpaceship;
  • Functions: must start with a lower case letter, and use CamelCase. boolean checkResult();
  • Enums: Preferred for readability over integer constants usually 
    class Directions {
    public enum Direction {
        NORTH,
        SOUTH,
        EAST,
        WEST
    }
}

  • What is method overloading vs method overriding?
    Method overloadingis methods that have the same name but have different number of parameters. Note: simply changing return type of two methods of the same name/signature will not work. The parameter must be different.
    Method overriding allows you to implement your own version of a method you inherited.
  • Heap Vs. Stack Memory
    Heap
    Grows/shrinks dynamically. When the "new" keyword is used, the object is placed here
    Stack
    Allocated per program. Fixed memory space. Local variables live here.
  • Shallow Vs. Deep COPY
    Shallow Copy
    Both objects changes as they are just references to each other
    Deep Copy
    Separate objects completely, can have different values
  • How do you run a unit test in Java?

    JUnit:


    • File to be tested (Hello.java): 
      public class Hello {

    public static int returnAnswerToEverything() {
        return 42;
    }
}
    • Test file (HelloTest.java): 
      import org.junit.Assert;
import org.junit.Before;
import org.junit.Test;
public class HelloTest {
    Hello h;
    @Before
    public void setUp() throws Exception {
        h = new Hello();
    }
    @Test
    public void testReturnsAnswer() {
        Assert.assertEquals(42, Hello.returnAnswerToEverything());
    }
