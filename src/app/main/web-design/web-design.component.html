    • Color Contrast
      The Web Content Accessibility Guidelines (WCAG) recommend a contrast ratio of at least 4.5:1 for normal text and 3:1 for large text. There are several standards for color contrast, but the most common is the WCAG 2.0 which requires a contrast ratio of at least 4.5:1 for normal text and 3:1 for large text. This ensures that text is readable and distinguishable from the background color. WCAG Level AAA requires a contrast ratio of at least 7:1 for normal text and 4.5:1 for large text.

    • Alt Text for Images

      Alt text is important for users who are visually impaired and rely on screen readers to navigate the web. Alt text provides a description of an image, allowing users to understand its content even if they can't see it.

      When adding alt text to an image, be sure to provide a concise and accurate description of the image. Avoid using generic phrases like "image" or "photo" and instead describe the content of the image in detail.

      Alt text should be added to all images on your website, including decorative images and images used as links. This will help ensure that your content is accessible to all users.

      Playwright Example

      Playwright can be used to validate the alt text of images on a page. This can be done by doing something such as:

      import { expect, test } from '@playwright/test';

test('all images have alt text', async ({ page }) => {
    // go to site
    await page.goto('https://www.nehsa.net');

    // get all images on the page
    const images = await page.$$('img');

    // check that each image has alt text
    for (const image of images) {
    const altText = await image.getAttribute('alt');
    expect(altText).not.toBeNull();
    expect(altText).not.toBe('');
    }
});
    • Keyboard Navigation

      Keyboard navigation is essential for users who are unable to use a mouse. To ensure that your website is accessible to all users, be sure to provide keyboard shortcuts and navigation options.

      When designing your website, consider the following:

      • Use semantic HTML elements to create a logical tab order
      • Provide keyboard shortcuts for common actions
      • Ensure that all interactive elements are accessible via the keyboard
      • Test your website using only the keyboard to ensure that all functionality is available

      By providing keyboard navigation options, you can make your website more accessible to users with disabilities and ensure that everyone can navigate your site easily.
    • Responsive Design

      Responsive design is essential for creating a website that looks good on all devices. With the increasing use of smartphones and tablets, it's important to ensure that your website is accessible on a variety of screen sizes.

      When designing a responsive website, consider the following:

      • Use media queries to adjust the layout of your website based on screen size
      • Optimize images and videos for different screen resolutions
      • Ensure that text is readable on all devices
      • Test your website on different devices to ensure that it looks good on all screen sizes

      By creating a responsive website, you can provide a better user experience for all visitors, regardless of the device they are using to access your site.

  • The "Display" Property Display property is used to change the layout of an element. The most common values are:
    grid
    Displays an element as a block-level grid container. Required to use properties such as:
    • grid-template-columns
    • grid-template-rows
    • grid-gap
    flex
    Displays an element as a block-level flex container. Required to use properties such as:
    • flex-direction
    • justify-content
    • align-items
    block
    Displays an element as a block-level element (like a div). It starts on a new line and takes up the full width available.
    inline
    This is the default value, which displays an element as an inline-level element (like a span). This means the element will not cause a line break and will only take up as much width as necessary.
    inline-block
    This displays an element as an inline-level block container, allowing the element to have a width and height.
    none
    Displays an element as if it wasn't there. The element will be invisible and not take up any space.
    inline-grid
    Displays an element as an inline-level grid container. An inline-level grid container is a grid container whose inline-level children participate in its grid formatting context.
  • Semantic HTML
    Semantic HTML is a coding practice that uses HTML elements to describe the meaning or purpose of content on a web page.

    Some common examples of semantic HTML elements include:
    • <header>: Defines a header for a document or section
    • <nav>: Defines a set of navigation links
    • <main>: Defines the main content of a document
    • <article>: Defines an article or section of content
    • <section>: Defines a section of content
    • <aside>: Defines content aside from the content it is placed in
    • <footer>: Defines a footer for a document or section


    By using semantic HTML elements, you can create web pages that are more accessible, easier to navigate, and better understood by search engines. By default, nothing is styled when you use these elements, so you can style them however you like.

    e.g. This:
    <nav>
    <div>item1</div>
    <div>item2</div>
    <div>item3</div>
</nav>

  • The "Position" Property Common positoning types and examples:
    block
    Block-level elements are those that take up the full width available and have a line break before and after the element.
    fixed
    Fixed elements are positioned relative to the browser window.
    Pro-tip: Using a fixed positioning without a top, left, right, or bottom property, the element will be placed in the top left corner the nearest positioned ancestor. This is similar to absolute position but can have an advantages:
    • It will not scroll with the page
    • It will not be affected by other elements
    • It will not be clipped
    absolute
    Absolute elements are positioned relative to the nearest positioned ancestor.
    relative
    Relative elements are positioned relative to their normal position. These are often floated left or right.
    sticky
    Sticky elements are positioned based on the user's scroll position. They are treated as relatively positioned until they reach a certain point and then they are treated as fixed.
  • Website Usability Metrics: Web Vitals (LCP, FID, and CLS) Website usability metrics are essential to ensure your website is user-friendly and provides a good user experience. This website's primary purpose is as a testbed to better understand and implement design best practices.

    Common ones are:
    Largest Contentful Paint (LCP)
    Measures loading performance. To provide a good user experience, LCP should occur within 2.5 seconds of when the page first starts loading. A slow LCP can result in a poor user experience and potentially drive visitors away from a website.
    Cumulative Layout Shift (CLS)
    CLS measures the visual stability of a webpage. It calculates the sum of all individual layout shift scores for any unexpected layout shifts that occur during a page's lifespan. A good aim for CLS is a score less than 0.1.
    First Input Delay (FID)
    FID is a measure of interactivity. It measures the time from when a user first interacts with a page (e.g., clicks something) to the time when the browser responds to that interaction. A good FID score is less than 100 milliseconds.

    • Leveraging counter-reset and counter-increment
      • Initialization: When the CSS rule containing counter-reset: list-level; is applied to an element, a counter named "list-level" is created and set to 0.
      • Incrementing: The counter-increment: list-level; property is used to increase the value of the "list-level" counter by 1 for each element where it's applied.
      • Accessing the counter: The counter() function can be used within the content property to display the current value of the counter.
      Example: 
      ul li::before {
    content: counter(list-level) " > ";
    counter-increment: list-level;
    position: relative;
    left: -1em;
}       

ul {
    list-style-type: circle;
    counter-reset: list-level;
}
    
ul ul {
    list-style-type: square;
    counter-increment: list-level;
}

      .pause-container:has(img) {

      fadeOut
      The name of the CSS animation function. In this case, it keeps something visible fully (no gradual disappearance) until the last 10% of the animation duration, then fades out.
      @keyframes fadeOut {
0% {
    opacity: 1;
}

90% {
    opacity: 1;

100% {
    opacity: 0;
}

      10s
      The duration of the animation. In this case, it takes 10 seconds to complete.
      ease-out
      The timing function. In this case, it starts fast and slows down at the end. Other options include:
      ease-in
      Starts slow and speeds up at the end
      ease-in-out
      Starts slow, speeds up, then slows down at the end
      linear
      Constant speed
      step-start
      Starts at the beginning of each step then jumps to the end
      step-end
      Starts at the end of each step then jumps to the beginning
      steps(4, end)
      4 steps, each starting at the end of the previous step
      cubic-bezier(0.1, 0.7, 1.0, 0.1)
      Custom cubic-bezier function

      cubic-bezier(x1, y1, x2, y2):
      Defines a curve based on four control points.
      (x1, y1) is the control point for the starting end of the curve.
      (x2, y2) is the control point for the ending end of the curve.
      Both (x1, y1) and (x2, y2) must be between 0 and 1.
      example:
      green/red
      The slow start and end simulate the compression and decompression of the ball, while the quick middle represents the bounce itself.
      animation: bounce 5s infinite cubic-bezier(0.25, 0.8, 0.25, 0.2);
      orange/blue

      animation: bounce 5s infinite cubic-bezier(0.4, 0, 0.2, 1);
      yellow/purple
      Object is decelerating for the stop.
      animation: bounce 5s infinite cubic-bezier(0.4, 0.1, 0.8, 1);
      fuchsia/teal
      Elastic/springy type effect
      animation: bounce 5s infinite cubic-bezier(0.1, 1, 0.9, 0);

      1
      Runs once
      2

      forwards
      Runs once and then stops at the end state
      backwards
      Runs once and then stops at the beginning state
      infinitely

  • Text and Element alignment
    CSS has too many ways to align stuff. I often find myself trying each until things finally work which is not the best way to do things. Here are some of the ways to align text and elements:
    text-align
    Aligns text within a container. Options include: left, right, center, and justify.
    vertical-align
    Aligns inline elements vertically. Options include: baseline, sub, super, top, text-top, middle, bottom, text-bottom, and length.
    line-height
    Specifies the height of a line.
    justify-content
    Aligns items horizontally and accepts the following values: flex-start, flex-end, center, space-between, space-around, and space-evenly.
    align-items
    Aligns items vertically and accepts the following values: stretch, flex-start, flex-end, center, baseline, and space-between.
    align-self
    Aligns a single item vertically and accepts the following values: auto, stretch, flex-start, flex-end, center, baseline, and space-between.
    place-content
    Aligns items both horizontally and vertically and accepts the following values: flex-start, flex-end, center, space-between, space-around, space-evenly, start, end, and stretch. Is shorthand for align-content and justify-content.
    place-items (do not use)
    Aligns items both horizontally and vertically and accepts the following values: start, end, center, stretch, flex-start, flex-end, self-start, self-end, baseline, first baseline, last baseline, safe center, unsafe center, legacy, and legacy center. Limiations and when to use: place-items is a shorthand for align-items and justify-items. It is not supported in Firefox, but it is supported in Chrome and Safari. It is not recommended to use place-items because it is not supported in all browsers. Instead, use align-items and justify-items separately.
    margin: auto;
    Centers an element horizontally and vertically. This is often used with flex or grid containers.

    margin: 0 auto; arrow_backHorizontally align

  • Fun with Flex(box) I've tried exclusively using CSS Grids over Flexbox for the longest time. When comparing the two, CSS Grids seem superior. I was mistaken, as the world continues using flex-style design.

    For the most part, CSS Grid is a superior method for controlling your page layout and responsive design considerations, but there are a few reasons to choose Flexbox:

    Flexbox excels at one-dimensional layouts, either horizontal or vertical. It is also easier to use and has a lower learning curve.

    Definitions
    Flexbox
    Flexbox is a one-dimensional layout method for laying out items in rows or columns. Items flex to fill additional space and shrink to fit into smaller spaces.
    Flex Container
    The parent element that contains the flex items.
    Flex Axis
    The main axis and the cross axis of a flex container.
    Main Axis
    Left-Right if row, Top-Bottom if column.
    Cross Axis
    The axis perpendicular to the main axis.
    Flex Item
    The children of a flex container.
    Flex Direction
    Defines the direction items are placed in the flex container. Options include row, row-reverse, column, and column-reverse.
    row
    Items are placed the same as the text direction.
    row-reverse
    Items are placed opposite to the text direction.
    column
    Items are placed top to bottom.
    column-reverse
    Items are placed bottom to top.
    Justify Content
    Aligns items on the main axis. The main axis is the axis the flex items are placed on. It goes from left to right in a row and from top to bottom in a column. Options include flex-start, flex-end, center, space-between, space-around, and space-evenly.
    flex-start
    Items align to the start of the container.
    flex-end
    Items align to the end of the container.
    center
    Items align at the center of the container.
    space-between
    Items display with equal spacing between them.
    space-around
    Items display with equal spacing around them.
    space-evenly
    Items display with equal spacing around them, including the space before the first and after the last item.
    Align Items
    Aligns items on the cross axis. Options include flex-start, flex-end, center, baseline, and stretch.
    flex-start
    Items align to the start of the cross axis.
    flex-end
    Items align to the end of the cross axis.
    center
    Items align at the center of the cross axis.
    baseline
    Items display at the baseline of the cross axis. The baseline is the line where the letters sit.
    stretch
    Items are stretched to fit the container.
    Align Content
    Aligns a flex container's lines within when there is extra space in the cross-axis. Options include flex-start, flex-end, center, space-between, space-around, and stretch.
    Flex Wrap
    Specifies whether flex items are forced onto one line or can wrap onto multiple lines. Options include nowrap, wrap, and wrap-reverse.
    Flex Grow
    Specifies how much a flex item will grow relative to the rest of the flex items.
    Flex Shrink
    Specifies how much a flex item will shrink relative to the rest of the flex items.
    Flex Basis
