      • Setup New Application
        1. npm init vue@latest
        2. npm install -g @vue/cli: Install CLI
        3. Configure Project Settings
        4. CD into new project directory and run:
          • npm install
          • npm run format (automatically format the codebase according to predefined style guidelines)
        5. We should now be able to start the new application: npm run dev
      • Create New Component
        • Create a new .vue file
        • Base code: 
          <template>
  <div> </div>
</template>

<script>
export default {
  name: 'MyComponent'
};
</script>
        • Import the component into the main application file: 
          import MyComponent from './MyComponent.vue';

new Vue({

  components: {

  }

      • Vue Application File Hierarchy
          • /
              package.json: Application metadata, package versions
              • /src
                • App.vue: Main component, often the root component
                • /stores: Global state of your application
                • /assets: Static files like images
                • /components: Reusable Vue components

        • HTML / TEMPLATE The template is where the HTML goes for this page:
          <template>
  HTML CONTENT
</template>


          Similar to Angular's ngFor/ngIf type options, Vue has v-for and v-if for providing dynamic data to the template.
          Operation Purpose Example Angular Equivalent
          Text Interpolation Display variable { { someVariable } } Curly braces should not have space between them, this is only to allow Angular to build. Same
          v-for Loop through an array 
          <li v-for="item in items" :key="item.id">
  { { item.text } }
</li>
          		ngFor
          v-if Conditional rendering 
          <div v-if="isTrue">This is true</div>
<div v-else>This is false</div>
          		ngIf
          v-model Two-way data binding. i.e. input field updates variable and variable updates input field 
          <input v-model="message" placeholder="Enter a message">
<p>The message is: { { message } }</p>
          		[(ngModel)]
          v-on Event handling. i.e. click, hover, etc.

          Shorthand for this using the @ symbol:

          <a v-on:click="go">...</a> === <a @click="go">...</a> 
          <button v-on:click="doSomething">Click me</button>
          		(click)="doSomething"
          v-bind Bind an attribute to a variable. This is better than using src= only because it allows for dynamic values. ImageSrc could be a variable that changes based on user input. 
          <img v-bind:src="imageSrc">
          		[src]
          v-html Render content as HTML. 
          <div v-html="htmlContent"></div>
          		Use innerHTML
        • SCRIPT Under the template area is the script area. This is where you define the scripting logic for the page. 

          <script>
  export default
    name: '',
    components: {}, All other components used in this component
    props: [], Properties/data passed to this component from parent
    data: function () {}, Variables used in this component
    computed: {}, Use computed property based on a data property for caching. The computed property will only update when the data
      property changes. If you use a method, it will update every time the page is rendered.
    watch: {}, Properties to watch for changes
    methods: {}, Methods used in this component
    mounted: function () {}, Code to run when component first 
      rendered on the page. This is akin to Angular's ngOnInit
  }
</script>
        • STYLE Styling is done here. There's a few important options availale to the style tag:
          • Adding scoped <style scoped> will scope the CSS to this component only. Without the scoped attribute, styling of this component will affect all components both above and below it in the heirarchy.
          • Adding lang="<OPTION>" will allow for SCSS styling. Options: sass, scss, less, stylus. 
          <style lang="scss">
  CSS/SCSS CONTENT
</style>

      • Passing Data Between Components Event order of operations:
        1. Parent's create
        2. Child's create
        3. Child's mounted
          • Parent's mounted
        • Props: Parent to Child (Not yet a working example) Parent:
          <template>
  <child-component :message="dataVar"></child-component> :message is the prop, dataVar is the variable being passed.
</template>

<script>
  export default
    data: function () { Data to pass to child component. The data: function () is a function that returns an object.
      return {
        dataVar: 'Hello World' Where the contents of dataVar are being set
      }
    }
</script>


          Child:
            
<template>
  <div>{ { data-var } }</div>
</template>

<script>
  export default
  Where we specify what is coming from the parent. Use kebab-case here for props. dataVar becomes data-var.
    props: {
      data-var: String 
    }
</script>
        • Emit (Events): Child to Parent (Not yet a working example) Data can be passed back from a child to the parent using the emit function:
          Child: this.$emit('data', 'Hello World');

          Parent:
          <template>
  <child-component @data="getData"></child-component>
  <div>{ { dataVar } }</div>
</template>

<script>
  export default
    data: function () {
      return {
        dataVar: ''
      }
    },
    methods: {
      getData: function (data) {
        this.dataVar = data;
      }
    }
</script>
        • Event Bus: Sibling to Sibling (Not yet a working example)
          • Event bus is a Vue instance that allows components to communicate with each other.
          • It is a global instance that can be imported into any component.
          • It is not recommended for large applications as it can become difficult to manage.


          Event Bus, setup as a separate component:
          <script>
  export const eventBus = new Vue();
</script>
                            
          <template>
  <button @click="emitEvent">Click me</button>
</template>

<script>
  import { eventBus } from './event-bus.js';
  export default  
    methods: {
      emitEvent: function () {
        eventBus.$emit('event', 'Hello World');
      }
    }
</script>
                            
          <template>
  <div>{ { dataVar } }</div>
</template>

<script>
  import { eventBus } from './event-bus.js';
  export default
    data: function () {
      return {                          
        dataVar: ''
      }
    },
    mounted: function () {
      eventBus.$on('event', (data) => {
        this.dataVar = data;
      });
    }
</script>
