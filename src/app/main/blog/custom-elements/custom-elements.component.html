Custom HTML Elements
August 29th, 2024
Jesse Stone

Although I see the allure of CSS frameworks, which focus on atomic/single-use utility classes. These frameworks allow you to add them to an element to change its styling. For example, in TailWind CSS, you add the "items-center" class to its parent if you wish to center an element.

This approach does have some limitations. For example, it's straightforward to end up with inconsistent styling because you've left one of many classes of an element.

Leveraging semantic and custom HTML elements where possible is a superior strategy. This approach offers reusable components with consistent styling, enhancing the overall user experience.

For example, instead of: <span class="code text-white font-bold">CODE</span>
I would create: <inline-code>CODE</inline-code>

This gives me two classes:
<code> - For multiline code blocks which I want to start on a new line
<inline-code> - For small code snippets that I wish to be on the same line
