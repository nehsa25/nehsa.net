Custom HTML Elements
August 29th, 2024
Jesse Stone

Although I see the allure of CSS frameworks, which focus on atomic, utility classes, you add them as needed to an element to change its styling. For example, in TailWind CSS, you add the "items-center" class to its parent if you wish to center an element.

This approach does have some limitations. For example, it's straightforward to end up with inconsistent styling because you've left one of many classes of an element.

Leveraging semantic and custom HTML elements where possible is a superior strategy. This approach offers reusable components with consistent styling, enhancing the overall user experience.

For example, instead of: <span class="code">CODE</span> I would create: <inline-code>CODE</inline-code>

This gives me two classes:
<code> - For multiline code blocks which I want to start on a new line <inline-code> - For small code snippets that I wish to be on the same line

Some frameworks, such as Angular, will not allow custom elements by default. To use them, do the following:
  • Add
e.g. 
    @Component({
    selector: 'app-angular',
    standalone: true,
    imports: [MatCardModule, CommentComponent, MatExpansionModule, MatIcon, RouterLink, CopyComponent],
    schemas: [CUSTOM_ELEMENTS_SCHEMA], // arrow_back Add this line
    templateUrl: './angular.component.html',
    styleUrl: './angular.component.scss'
    })
