Custom HTML Elements
September 1st, 2024
Jesse Stone

Although I see the allure of CSS frameworks, which focus on atomic/single-use utility classes (they allow you to add them to an element to change its styling).

For example, in TailWind CSS, you can add the "items-center" class to a parent element to center the elements within it. This approach does have some limitations. For example, it's common to end up with inconsistent styling because you've left one-of-many classes off of an element.

Leveraging semantic and custom HTML elements where possible is a superior strategy. This approach offers reusable components with consistent styling, enhancing the overall user experience.

For example, instead of: <span class="code text-white font-bold">CODE</span>
I would create: <inline-code>CODE</inline-code>

Doing this gives me two classes:
<code> - For multiline code blocks which I want to start on a new line
<inline-code> - For small code snippets that I wish to be on the same line
